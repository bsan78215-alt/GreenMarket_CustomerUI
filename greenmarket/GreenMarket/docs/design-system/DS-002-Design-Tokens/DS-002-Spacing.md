## 1. Назначение документа (Purpose)

Определяет шкалу отступов (Spacing Tokens) и правила их использования в GreenMarket Customer UI.

## 2. Область действия (Scope)

Входит: базовая единица, шкала отступов, рекомендуемое применение.
Не входит: сеточная система макета, брейкпоинты (см. DS-003 Foundations / Grid).

## 3. Material Design Baseline

GreenMarket использует 4dp-сетку Material Design 3: все значения кратны 4, семантическая модель токенов — стандартный подход MD3. Если не определено иное, применяются рекомендации MD3.

## 4. GreenMarket Extensions (Расширения)

### 4.1 Product Principles Mapping

| Принцип | Следствие для Spacing |
| :--- | :--- |
| **White Space First** | Пространство — основной способ разделения информации, а не разделительные линии/рамки. |
| **Fast Perception** | Отступы помогают пользователю быстро считать визуальную структуру экрана. |
| **Compare Naturally** | Одинаковые группы элементов имеют одинаковые интервалы на всех экранах. |

### 4.2 Шкала GreenMarket

- **Space 0**: 0 dp
- **Space 1**: 4 dp
- **Space 2**: 8 dp
- **Space 3**: 12 dp
- **Space 4**: 16 dp
- **Space 5**: 20 dp
- **Space 6**: 24 dp
- **Space 8**: 32 dp
- **Space 10**: 40 dp
- **Space 12**: 48 dp
- **Space 16**: 64 dp

### 4.3 Рекомендуемое применение

- Интервал между связанными элементами: **Space 2** (8 dp)
- Интервал между строками формы: **Space 3** (12 dp)
- Padding карточки: **Space 4** (16 dp)
- Внешний отступ экрана: **Space 4** (16 dp)
- Интервал между секциями: **Space 6** (24 dp)
- Большой разделитель: **Space 8** (32 dp)

## 5. Ограничения (Constraints)

- Смешение нескольких пространственных систем не допускается.
- Использование отрицательных отступов допускается только по отдельному архитектурному решению.
- Уменьшение отступов ради размещения большего количества информации (за счет "воздуха") не допускается.
