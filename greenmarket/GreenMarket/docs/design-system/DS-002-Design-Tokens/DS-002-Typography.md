## 1. Назначение документа (Purpose)

Определяет типографические токены и правила их использования в GreenMarket Customer UI.

## 2. Область действия (Scope)

Входит: типографические роли, размеры, правила использования.
Не входит: выбор семейств шрифтов (см. ADR-0003), локализация, tone of voice (см. DS-003, DS-007).

## 3. Material Design Baseline

GreenMarket использует типографическую шкалу Material Design 3 без изменений — роли Display/Headline/Title/Body/Label с их стандартными значениями Font Size / Line Height / Weight / Letter Spacing идентичны MD3 Type Scale.

Семейства шрифтов зафиксированы архитектурным решением ADR-0003: Roboto (Android), SF Pro (iOS), Inter/системный стек (Web), Noto Sans (fallback).

## 4. GreenMarket Extensions (Расширения)

### 4.1 Product Principles Mapping

| Принцип (DS-001) | Следствие для типографики |
| :--- | :--- |
| **Content First** | Текст всегда в приоритете перед декоративными графическими элементами. |
| **Fast Perception** | Визуальная иерархия экрана должна однозначно считываться пользователем за 2–3 секунды. |
| **Sunlight First** | Размеры и насыщенность шрифтов обеспечивают надежную читаемость на улице. |
| **Compare Naturally** | Одинаковые данные (цена, рейтинг, расстояние) имеют одинаковые роли во всех карточках каталога. |

### 4.2 Ограничения GreenMarket поверх MD3

- Display не используется в основных пользовательских сценариях Stage 1 (только для маркетинговых или приветственных экранов).
- На одном экране допускается использование не более **4 типографических ролей** одновременно.
- Числовые значения одного типа (цена, количество, рейтинг) всегда используют один и тот же токен во всей системе.
- Выравнивание по умолчанию — по левому краю; выравнивание по ширине (justify) не используется.
- Минимальный размер для пользовательского контента — Body Small. Меньший размер — только для служебной информации с отдельным обоснованием.

### 4.3 Роли и типичное применение

- **Headline**: Заголовки экрана каталога, продавца, оформления заказа, поиска.
- **Title**: Заголовки в Bottom Sheet, карточках, секциях, диалогах, списках.
- **Body**: Основной контентный текст (Body Large рекомендуется как базовый размер по умолчанию).
- **Label**: Кнопки, бейджи, Chips, элементы Navigation Bar.

## 5. Ограничения (Constraints)

- Произвольные размеры шрифта в компонентах не допускаются — используются строго токены.
- Создание новых уникальных текстовых ролей внутри компонентов запрещено.
- Использование декоративных шрифтов в Customer UI не допускается.
