# GM-UX-006. Product Card — Требования на реализацию UX Stage 1

**Статус:** Approved

## 1. Цель
Привести экран Product Card в соответствие с утверждёнными UX-артефактами GreenMarket Stage 1.
Экран предоставляет пользователю полную информацию о выбранном товаре и связанном с ним продавце.
Реализация должна соответствовать:
* PRD GreenMarket Stage 1;
* GM-010 Stage 1 Model Mapping;
* Seller_Profile.md (PR-009);
* Platform Interface.

## 2. Архитектурное основание
Архитектурным основанием экрана являются доменные модели:
* Product;
* SellerProduct;
* Seller Profile.

Для отображения местоположения продавца используются связанные данные:
* Market (через marketId).

Экран не использует техническую сущность Seller.

## 3. Назначение экрана
Экран предназначен для ознакомления пользователя с выбранным товаром.
Пользователь должен иметь возможность:
* просмотреть фотографии товара;
* ознакомиться с информацией о товаре;
* определить продавца товара;
* перейти к карточке продавца.

Экран не предназначен для оформления покупки.

## 4. Используемые данные
* Из **Product** используются: название, описание, фотографии, характеристики товара.
* Из **SellerProduct** используются: сведения о публикации товара продавцом.
* Из **Seller Profile** используются: name, photo или logo, shortDescription.
* Из **Market** используются: name, address.

Использование других доменных сущностей не допускается.

## 5. Отображение товара
Для товара допускается отображение:
* фотографии;
* названия;
* описания;
* характеристик.

При отсутствии отдельных данных соответствующие области не отображаются.

## 6. Отображение продавца
Для продавца отображается информация, необходимая для перехода к его карточке.
Минимально отображается:
* название продавца.

Дополнительно допускается отображение:
* фотография или логотип;
* краткое описание;
* название рынка;
* ряд;
* место.

Используются данные Seller Profile и Market.

## 7. Переход в Seller Card
Пользователь должен иметь возможность открыть экран: **Seller Card**.
Переход выполняется только через область отображения продавца.
Выбор информации о продавце не должен конфликтовать с просмотром информации о товаре.

## 8. Состояния экрана
Экран должен поддерживать следующие состояния:
* **Загрузка**: Получение информации о товаре и продавце.
* **Основное состояние**: Отображается полная информация о товаре.
* **Частичная информация**: Если отдельные сведения о продавце отсутствуют, отображаются только доступные данные.
* **Ошибка**: При невозможности получить данные отображается сообщение об ошибке. Пользователь может выполнить повторную загрузку.

## 9. Поведение пользователя
Поддерживаются действия:
* просмотр фотографий товара;
* просмотр информации о товаре;
* просмотр информации о продавце;
* переход в Seller Card;
* возврат на предыдущий экран.

Другие пользовательские сценарии Stage 1 отсутствуют.

## 10. Архитектурные ограничения
При реализации запрещается:
* изменять Domain Models;
* изменять Seller Profile;
* изменять Product;
* изменять SellerProduct;
* изменять Market;
* изменять Platform Interface;
* изменять Builder;
* изменять Adapter;
* изменять Presentation;
* изменять Runtime;
* реализовывать корзину;
* реализовывать оформление заказа;
* реализовывать способы покупки;
* реализовывать оплату;
* реализовывать доставку;
* реализовывать отзывы;
* реализовывать рейтинги;
* реализовывать чат.

## 11. Критерии готовности
Экран считается реализованным при выполнении следующих условий:
1. Реализован экран Product Card, соответствующий UX Stage 1.
2. Отображается информация о товаре.
3. Отображается информация о продавце на основе Seller Profile.
4. Используются данные Market для отображения местоположения продавца.
5. Реализован переход в Seller Card.
6. Поддерживаются состояния загрузки, частичной информации и ошибки.
7. Отсутствуют функции последующих Stage.
8. Архитектура Platform Interface не изменена.
