# GM-UX-007. Search — Требования на реализацию UX Stage 1

**Статус:** Approved

## 1. Цель
Реализовать экран Search как производный экран Customer UI GreenMarket Stage 1.
Экран предоставляет единый механизм поиска по цифровому каталогу GreenMarket и обеспечивает поиск товаров, товарных групп и продавцов независимо от экрана, с которого пользователь начал поиск.
Реализация должна соответствовать:
* PRD GreenMarket Stage 1;
* GM-010 Stage 1 Model Mapping;
* Seller_Profile.md (PR-009);
* Platform Interface.

## 2. Архитектурное основание
Архитектурным основанием экрана являются доменные модели:
* Product;
* ProductGroup;
* SellerProduct;
* Seller Profile.

Для отображения информации о местоположении продавца допускается использование связанных данных:
* Market (через marketId).

Экран не использует техническую сущность Seller.

## 3. Назначение экрана
Экран предназначен для поиска объектов цифрового каталога.
Пользователь должен иметь возможность:
* искать товары;
* искать товарные группы;
* искать продавцов;
* переходить к найденным объектам.

Экран не предназначен для оформления покупки.

## 4. Используемые данные
* **Product**: название, описание (если участвует в поиске), основные сведения для отображения результата.
* **ProductGroup**: название группы.
* **SellerProduct**: сведения, необходимые для формирования результатов поиска товаров.
* **Seller Profile** (только разрешённые поля): name, shortDescription.
* **Дополнительно для продавца**: photo или logo, Market.name, row, place.

## 5. Поисковый запрос
Экран предоставляет пользователю возможность ввести поисковый запрос. Настоящий документ не определяет техническую реализацию поиска (механизм, алгоритмы, полнотекстовый или семантический поиск), а задает только пользовательский сценарий отображения результатов.

## 6. Отображение результатов
Результаты могут содержать:
* **Товар** (фотография, название, продавец) -> открывает **Product Card**.
* **Продавец** (фото/логотип, название, краткое описание, название рынка) -> открывает **Seller Card**.
* **Товарная группа** (название группы) -> открывает **Catalog** с отображением выбранной группы.

## 7. Состояния экрана
* **Исходное состояние**: Запрос отсутствует, результаты не отображаются.
* **Выполнение поиска**: Отображается состояние поиска.
* **Основное состояние**: Отображаются найденные результаты.
* **Ничего не найдено**: Отображается соответствующее сообщение.
* **Ошибка**: Сообщение об ошибке с возможностью повторить попытку.

## 8. Поведение пользователя
Поддерживаются действия:
* ввод, изменение и очистка поискового запроса;
* выбор найденного товара, продавца или товарной группы;
* возврат на предыдущий экран.

## 9. Архитектурные ограничения
При реализации строго запрещается:
* изменять любые Domain Models, Platform Interface, Builder, Adapter, Presentation, Runtime;
* реализовывать корзину, оформление заказа, оплату, доставку;
* реализовывать рекомендации, историю поиска и сохранённые запросы.

## 10. Критерии готовности
1. Реализован самостоятельный экран Search.
2. Выполняется поиск по товарам, группам и продавцам.
3. Настроены переходы: товар -> Product Card, продавец -> Seller Card, группа -> Catalog.
4. Поддерживаются все 5 состояний экрана.
5. Архитектура Platform Interface не изменена.
