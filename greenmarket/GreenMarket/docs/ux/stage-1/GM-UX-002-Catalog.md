# UX-002. Catalog

**GreenMarket Stage 1 — Screen 2**

## 1. UX-описание

### Назначение

Экран Catalog предоставляет покупателю возможность просматривать цифровой каталог товаров GreenMarket независимо от их расположения на карте.

Основная задача экрана — помочь пользователю найти интересующие товары, ознакомиться с ассортиментом и перейти к продавцу, предлагающему выбранный товар.

Экран не предназначен для оформления покупки.

### Цель пользователя

Пользователь хочет:

- просматривать ассортимент товаров;
- переходить между товарными группами;
- быстро находить интересующие товары;
- открыть карточку товара;
- перейти к продавцу выбранного товара.

### Основной сценарий

1. Пользователь открывает экран Catalog.
2. Просматривает товарные группы.
3. Выбирает интересующую группу.
4. Просматривает список товаров.
5. Открывает карточку товара.
6. При необходимости переходит к карточке продавца (клик на название продавца).

### Альтернативные сценарии

**Поиск** — пользователь начинает поиск → переходит на экран Search.

**Просмотр карты** — пользователь переключается на экран Map.

**Просмотр списка продавцов** — пользователь переключается на экран Seller List.

**Повторная загрузка** — при ошибке пользователь повторяет загрузку каталога.

## 3. Семантическая модель

### Область поиска

| Параметр | Значение |
|---|---|
| Назначение | Начало поиска товаров или продавцов |
| Отображаемая информация | поисковый запрос |
| Приоритет | Высокий |
| Появление | Всегда |
| Скрытие | Никогда |
| Фиксированная область | Да |
| Поведение при нехватке места | Не изменяется |
| Загрузка | Доступна сразу |
| Пустое состояние | Пустое поле |
| Ошибка | Не отображается отдельно |

### Область товарных групп

| Параметр | Значение |
|---|---|
| Назначение | Предоставить быстрый выбор категории товаров |
| Отображаемая информация | перечень товарных групп |
| Приоритет | Высокий |
| Появление | После загрузки каталога |
| Скрытие | При отсутствии данных |
| Фиксированная/прокручиваемая | Фиксированная |
| Поведение при нехватке места | Отображается доступная часть списка с возможностью просмотра остальных |
| Загрузка | Показывается состояние загрузки |
| Пустое состояние | Нет доступных групп |
| Ошибка | Не удалось загрузить группы |

### Область списка товаров

| Параметр | Значение |
|---|---|
| Назначение | Отобразить товары выбранной группы |
| Отображаемая информация | для каждого товара: фотография; название; краткое описание; обязательное поле продавца (`sellerName`) |
| Приоритет | Основной |
| Появление | После выбора группы или открытия каталога |
| Скрытие | Во время первоначальной загрузки |
| Фиксированная/прокручиваемая | Прокручиваемая |
| Поведение при нехватке места | Количество отображаемых товаров определяется доступным пространством |
| Загрузка | Отображается состояние загрузки |
| Пустое состояние | Товары отсутствуют |
| Ошибка | Не удалось загрузить каталог |

## 4. Поведенческая модель

**Область поиска** — появляется при открытии экрана; обновляется при вводе текста; открывает экран Search.

**Область товарных групп** — появляется после загрузки данных; обновляется при выборе другой группы; изменяет состав отображаемых товаров. Состояния: загрузка; отображение; пустой список; ошибка.

**Область списка товаров** — появляется после получения данных; обновляется при смене товарной группы; обновляется после нового поискового запроса; 
* выбор товара (клик по области карточки, исключая имя продавца) открывает Product Card; 
* выбор продавца (клик строго по области `sellerName`) вызывает существующее действие `OPEN_SELLER` и открывает Seller Card. 
Состояния: загрузка; отображение списка; отсутствие товаров; ошибка загрузки; повторная загрузка.
