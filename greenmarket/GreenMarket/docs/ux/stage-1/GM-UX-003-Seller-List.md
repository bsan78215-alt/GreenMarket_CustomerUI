# UX-003. Seller List

**GreenMarket Stage 1 — Screen 3**

## 1. UX-описание

### Назначение

Экран Seller List предоставляет пользователю список всех активных продавцов, входящих в цифровой каталог GreenMarket.

Экран предназначен для случаев, когда пользователю удобнее выбирать продавца по списку, а не по расположению на карте.

Основной акцент сделан на сравнении продавцов и быстром переходе к их карточкам.

### Цель пользователя

Пользователь хочет:
- просмотреть список всех доступных активных продавцов;
- сравнить продавцов по краткой информации и местоположению;
- открыть карточку выбранного продавца.

### Основной сценарий

1. Пользователь открывает экран Seller List.
2. Просматривает список продавцов (из модели `Seller Profile`, имеющих статус `ACTIVE` и привязку к `Market`).
3. Выбирает продавца из списка.
4. Переходит на экран `Seller Card`.

### Альтернативные сценарии

**Просмотр карты** — пользователь переключается на экран Map.

**Просмотр каталога** — пользователь переключается на экран Catalog.

**Поиск** — пользователь начинает поиск → переходит на экран Search.

**Повторная загрузка** — при ошибке пользователь выполняет повторную загрузку списка.

## 3. Семантическая модель

### Область поиска

| Параметр | Значение |
|---|---|
| Назначение | Начало поиска товара или продавца |
| Отображаемая информация | поисковый запрос |
| Приоритет | Высокий |
| Появление | Всегда |
| Скрытие | Никогда |
| Фиксированная/прокручиваемая | Фиксированная |
| Загрузка | Доступна сразу |
| Пустое состояние | Пустое поле |
| Ошибка | Не отображается отдельно |

### Область списка продавцов

| Параметр | Значение |
|---|---|
| Назначение | Предоставить пользователю перечень активных продавцов |
| Отображаемая информация | Для каждого продавца. Из `Seller Profile`: `name`, `shortDescription`, `photo`/`logo`, `workingHours`, `row`, `place`. Из `Market`: `name` (название рынка), `address`. |
| Приоритет | Основной |
| Появление | После загрузки данных |
| Скрытие | Во время первоначальной загрузки |
| Фиксированная/прокручиваемая | Прокручиваемая |
| Поведение при нехватке места | Количество отображаемых продавцов определяется доступным пространством |
| Загрузка | Отображается состояние загрузки |
| Пустое состояние | Активные продавцы отсутствуют |
| Ошибка | Не удалось загрузить список продавцов |

## 4. Поведенческая модель

**Область поиска** — открывает экран Search при вводе запроса.

**Область списка продавцов** — появляется после получения данных; обновляется при изменении результатов поиска; выбор продавца (клик по элементу списка) открывает экран `Seller Card`. Поддерживает состояния: загрузка; отображение списка; пустой список; ошибка загрузки; повторная загрузка.
