# GM-UX-004. Seller List & Seller Card — Системные и UX требования (Stage 1)

**Статус:** Approved  
**Версия:** 1.0  
**Контекст:** GreenMarket Stage 1 — Customer UI  

---

## 1. Назначение и бизнес-цели

Документ объединяет требования к двум ключевым элементам интерфейса покупателя (Customer UI) в рамках Stage 1:
1. **Экран Seller List (Список продавцов)** — альтернативное текстовое представление продавцов, находящихся на карте (Map), предназначенное для быстрого поиска, сравнения и перехода к конкретному продавцу.
2. **Экран Seller Card (Карточка продавца)** — детальное представление выбранного продавца и его ассортимента, являющееся конечной точкой просмотра в Stage 1 (без функций покупки).

---

## 2. Архитектурные основания и ограничения

### Доменная модель и данные
Реализация строго базируется на утвержденных Domain Models. Использование иных сущностей запрещено.

* **Seller Profile (из PR-009):**
    * `name` — название продавца.
    * `shortDescription` — краткое описание.
    * `photo` / `logo` — изображение.
    * `workingHours` — часы работы.
    * `row` — ряд на рынке.
    * `place` — место на рынке.
    * `status` — статус (в список допускаются только продавцы со статусом **ACTIVE** и полностью заполнившие профиль).
* **Market (связанная сущность через `marketId`):**
    * `name` — название рынка.
    * `address` — физический адрес рынка.

### Системные ограничения Stage 1
* **Запрещено:** изменять Domain Models, Seller Profile, Market, Platform Interface, Builder, Adapter, Presentation, Runtime.
* **Запрещено:** использовать Bottom Sheet как самостоятельный бизнес-экран.
* **Запрещено:** реализовывать корзину, оформление заказа, оплату и доставку.
* **Запрещено:** добавлять любые пользовательские сценарии из последующих стадий (Stage 2+).

## 3. Экран 1: Seller List (Список продавцов)

### 3.1. Поведение и логика отображения
* Каждый активный продавец (`ACTIVE`) отображается в списке строго один раз.
* Элемент списка содержит:
    * Логотип/фотографию продавца;
    * Название продавца;
    * Краткое описание;
    * Расположение: Название рынка (`Market.name`), Ряд (`row`), Место (`place`);
    * Режим работы (`workingHours`).
* Дополнительная информация (например, описание или конкретное место) скрывается, если значение не заполнено в БД.

### 3.2. Навигационная модель
* **Выбор продавца:** Тап по карточке продавца открывает экран **Seller Card**.
* **Переключение экранов:** Должна быть обеспечена навигация на основные экраны:
    * `Map` (Карта);
    * `Catalog` (Общий каталог).
* **Поиск:** При вызове поиска происходит бесшовный переход на глобальный экран `Search`. Собственная поисковая логика внутри Seller List отсутствует.

### 3.3. Состояния экрана
1. **Загрузка (Loading):** Идет получение списка продавцов через Platform Interface. Отображается плейсхолдер/спиннер.
2. **Основное состояние (Success):** Отображается прокручиваемый список карточек продавцов.
3. **Пустое состояние (Empty):** Если активных продавцов нет, выводится информационное сообщение (например: *"Нет доступных продавцов"*).
4. **Ошибка (Error):** При сбое сети/сервера выводится сообщение об ошибке и кнопка *"Повторить попытку"*.

## 4. Экран 2: Seller Card (Карточка продавца)

### 4.1. UX-Описание и сценарии
Экран является тупиковой веткой исследования для Stage 1. Оформить заказ отсюда невозможно.

* **Основной сценарий:**
    1. Пользователь переходит из Seller List или Map в карточку продавца.
    2. Просматривает блок информации о продавце.
    3. Листает список (ассортимент) товаров продавца.
    4. Выбирает интересующий товар и переходит на экран **Product Card** (Карточка товара).
* **Альтернативные сценарии:**
    * *Возврат:* Переход назад на предыдущий экран (Seller List или Map).
    * *Поиск:* Переход на экран `Search`.
    * *Повторная загрузка:* Инициация повторного запроса при ошибке сети.

### 4.2. Wireframe экрана Seller Card

```
+----------------------------------------------------+
|               [ < Назад ]       [ Поиск ]          |
+----------------------------------------------------+
|                Карточка продавца                   |
+----------------------------------------------------+
| Название продавца (Seller Profile.name)            |
| Краткое описание (shortDescription)                |
| Местоположение (Market.name, Ряд, Место)           |
| Режим работы / Дополнительная информация           |
+----------------------------------------------------+
| Товары продавца                                    |
| -------------------------------------------------- |
| [ Фото ]  Название товара                          |
|           Краткое описание товара                  |
| -------------------------------------------------- |
| [ Фото ]  Название товара 2                        |
|           Краткое описание товара 2                |
| -------------------------------------------------- |
+----------------------------------------------------+
```

### 4.3. Семантическая модель Seller Card

#### Блок А: Информация о продавце (Фиксированная область)
* **Назначение:** Идентификация и базовая информация о продавце.
* **Состав данных:** Название, описание, местоположение на рынке, режим работы.
* **Приоритет:** Высокий.
* **Появление:** После загрузки данных продавца.
* **Скрытие:** Во время загрузки.
* **Поведение при нехватке места:** Текст дополнительной информации усекается (ellipsis).
* **Поведение при загрузке:** Отображается состояние загрузки.
* **Пустое состояние:** Данные отсутствуют.
* **Состояние ошибки:** Не удалось загрузить сведения.

#### Блок Б: Ассортимент продавца (Прокручиваемая область)
* **Назначение:** Показ списка товаров данного продавца.
* **Состав данных (для каждого товара):** Фотография, название товара, краткое описание.
* **Приоритет:** Основной.
* **Появление:** После загрузки данных.
* **Скрытие:** Во время загрузки.
* **Поведение при нехватке места:** Количество отображаемых товаров определяется доступным пространством.
* **Поведение при загрузке:** Отображается состояние загрузки.
* **Пустое состояние:** У продавца отсутствуют опубликованные товары.
* **Состояние ошибки:** Не удалось загрузить ассортимент.

## 6. Критерии приемки (Definition of Done)

1. Экран **Seller List** создан как независимый полноэкранный View (не Bottom Sheet).
2. На экране **Seller List** корректно выводятся только продавцы со статусом `ACTIVE`.
3. Корректно связываются данные `Seller Profile` и `Market` (отображается адрес и название рынка для продавца).
4. Навигационные триггеры перехода на `Map`, `Catalog` и `Search` функционируют согласно архитектуре Presentation.
5. Экран **Seller Card** успешно принимает ID продавца, загружает детальные данные и ассоциированный список товаров.
6. В **Seller Card** отсутствует любая возможность добавления товара в корзину (нет кнопок «Купить», «В корзину», ценников с селектором количества).
7. Все состояния (Загрузка, Пусто, Ошибка) визуально соответствуют гайдлайнам Stage 1 и поддерживают ручной перезапуск (Retry).
