## Отличие от v1/v2

v1 и v2 — это архитектура и БД подсистемы каталога (продавец → публикация → единый каталог), уже сведённые в `../../docs/architecture/`.

**Этот пакет — новая, отдельная тема: Customer UI (покупатель).** Все 19 чатов озаглавлены в ChatGPT одинаково — «Миграция Customer UI» — и представляют собой единую последовательную серию документов «ТЗ-NNN», написанных подряд в одном стиле (ChatGPT предлагает следующий документ в конце каждого сообщения, следующая ссылка его раскрывает). Ни разу не пересекается с продуктовой моделью каталога/публикации из v1/v2 — центральная сущность здесь Purchase Intent (покупка), а не Product/Catalog.

Это черновой материал уровня продуктового ТЗ, ещё не сверенный и не сведённый в `docs/architecture/`. Считать неавторитетным до отдельного шага синтеза (как это было сделано для v1 → v2 → `docs/architecture/`).

## Архитектурное ограничение

Platform Core уже является частью архитектуры проекта.

FSM Engine в текущую версию не входит и будет внедрён позже.

Все документы Customer UI должны проектироваться так, чтобы переход от текущей CRUD-реализации к FSM Engine потребовал замены реализации сервисов, но не изменения:

- Customer UI;
- ViewModel;
- User Flows;
- Business Rules;
- Action Catalog;
- Event Catalog.
