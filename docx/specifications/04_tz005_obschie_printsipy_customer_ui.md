## 1. Основной принцип

Customer UI представляет собой единое рабочее пространство.

Пользователь не должен ощущать переходов между отдельными экранами.

Все действия выполняются в пределах одного интерфейса.

## 2. Карта

Карта является основой интерфейса. Она:

- никогда не закрывается полностью;
- не заменяется другими экранами;
- сохраняет положение камеры;
- сохраняет масштаб;
- сохраняет выбранные объекты.

Все остальные элементы работают поверх карты.

## 3. Bottom Sheet

Bottom Sheet является основной рабочей областью. Он содержит: корзину; домашние запасы; варианты покупки; результаты поиска; карточку продавца; карточку товара.

Bottom Sheet не является отдельным экраном.

## 4. Контекст

Интерфейс всегда находится в одном текущем контексте. Например: обзор рынка; подготовка покупки; просмотр варианта покупки; просмотр продавца; просмотр товара.

Контекст определяет содержимое Bottom Sheet. Карта при этом остаётся той же.

## 5. Непрерывность работы

Любое действие пользователя должно менять только необходимую часть интерфейса.

Не допускаются: полная перерисовка главного экрана; возврат к начальному состоянию; потеря положения карты; потеря выбранного варианта покупки.

## 6. Сохранение состояния

При возврате назад должны сохраняться: положение карты; масштаб; выбранный вариант покупки; положение Bottom Sheet; результаты поиска; фильтры; открытая корзина.

Пользователь должен продолжить работу с того же места.

## 7. Работа с сетью

Интерфейс не должен блокироваться во время получения данных.

Пользователь может продолжать: перемещать карту; открывать карточки; изменять корзину.

Данные обновляются по мере поступления.

## 8. Постепенное обновление

Любое изменение должно обновлять только связанные данные.

Например: изменились домашние запасы — не должны заново загружаться карта, каталог, продавцы. Пересчитывается только текущая покупка.

## 9. Приоритет информации

Если информации слишком много, отображается только наиболее полезная пользователю.

Приоритет определяется: текущей покупкой; любимыми продавцами; расстоянием; актуальностью данных; пользовательскими настройками.

## 10. Работа без регистрации

Все функции подготовки покупки доступны без регистрации.

Регистрация требуется только для: синхронизации; резервного хранения; использования на нескольких устройствах.

## 11. Работа с ошибками

Если часть данных недоступна, интерфейс продолжает работать.

Например: нет связи с сервером остатков — не перестают работать карта, корзина, поиск, домашние запасы.

Пользователь получает информацию о том, какие данные могут быть неактуальны.

## 12. Скорость работы

Интерфейс должен создавать ощущение мгновенного отклика.

Если операция занимает значительное время, пользователь должен видеть, что система продолжает работу.
