## Назначение

Главный экран является единственной точкой входа покупателя в GreenMarket.

Вся работа пользователя выполняется в пределах этого экрана.

Переходы на отдельные полноэкранные страницы не допускаются, кроме системных экранов (авторизация, настройки и т.п.).

## 1. Общая структура

```
┌──────────────────────────────────────────────┐
│ Status Bar                                    │
├──────────────────────────────────────────────┤
│                                                │
│                                                │
│                    MAP                        │
│                                                │
│                                                │
├──────────────────────────────────────────────┤
│                                                │
│              Bottom Sheet                     │
│                                                │
└──────────────────────────────────────────────┘
```

Карта всегда остаётся видимой. Bottom Sheet никогда полностью не закрывает карту.

## 2. Layout

**Status Bar** — стандартная системная панель, не используется для размещения элементов приложения.

**Map Area** — занимает всё пространство между Status Bar и Bottom Sheet. Карта является главным рабочим пространством приложения.

**Bottom Sheet** — располагается поверх карты, всегда присутствует, никогда полностью не скрывается.

## 3. Начальное состояние

После открытия приложения пользователь сразу видит:

**На карте:** своё текущее местоположение; ближайших продавцов; любимых продавцов; продавцов возле сохранённых мест; продавцов вдоль регулярных маршрутов.

**Bottom Sheet** в свёрнутом состоянии отображает: строку поиска; активную корзину; кнопку выбора корзины; краткую информацию о найденных вариантах покупки (если уже рассчитаны).

## 4. Bottom Sheet

Имеет четыре фиксированных состояния:

- **Hidden** — используется только системой, пользователь вручную открыть не может.
- **Collapsed** — высота ~20–25% экрана. Видно: строку поиска; название активной корзины; количество товаров; кнопку «Подобрать покупку».
- **Half** — высота ~55%. Используется для: результатов поиска; вариантов покупки; списков продавцов.
- **Expanded** — высота ~90%. Используется для: карточки продавца; карточки товара; домашнего каталога; домашних запасов.

## 5. Карта

По умолчанию показывает: продавцов; местоположение пользователя; значимые места; маршрут (если выбран).

Не показывает: каталог товаров; всплывающие окна; модальные карточки. Вся дополнительная информация отображается через Bottom Sheet.

## 6. Поиск

Строка поиска всегда доступна. Нажатие на неё: переводит Bottom Sheet в состояние Half; открывает поиск; не изменяет положение карты.

## 7. Выбор корзины

При нажатии открывается список: типовых корзин; пользовательских корзин; последней использованной корзины.

После выбора: пересчитывается Purchase Intent; варианты покупки становятся неактуальными; карта остаётся на месте.

## 8. Подбор покупки

После нажатия кнопки «Подобрать покупку» происходит: запуск расчёта Purchase Optimizer; отображение индикатора процесса; сохранение возможности пользоваться картой; отображение вариантов покупки после завершения расчёта.

## 9. Выбор варианта покупки

После выбора варианта карта автоматически: подсвечивает участвующих продавцов; строит маршрут; показывает покрытие покупки.

Bottom Sheet переходит в состояние Half.

## 10. Открытие продавца

Нажатие на продавца не изменяет карту. Bottom Sheet раскрывается до Expanded. Загружается карточка продавца.

## 11. Открытие товара

Карточка товара открывается поверх карточки продавца. Используется внутренний стек Bottom Sheet. Карта остаётся без изменений.

## 12. Кнопка Back

Работает со стеком интерфейса:

```
Товар → Продавец → Варианты покупки → Поиск → Главный экран → Выход
```

Карта при этом не перестраивается.

## 13. Состояния загрузки

Допускаются независимые загрузки: карты; продавцов; каталога; фотографий; вариантов покупки. Отказ одной загрузки не блокирует остальные.

## 14. Пустые состояния

Отдельно определяются: не найдено продавцов; не найден товар; нет вариантов покупки; нет корзин; нет домашних запасов. Для каждого — отдельный экран внутри Bottom Sheet.

## 15. Работа офлайн

Если отсутствует соединение, доступны: последняя корзина; домашние запасы; ранее просмотренные карточки; карта с кэшированными данными. Недоступные действия должны быть явно обозначены.

## 16. Анимации

Все переходы выполняются без смены Activity или страницы. Допускаются только: изменение высоты Bottom Sheet; анимация открытия карточек; анимация маршрута; плавное центрирование карты.

## 17. Производительность

Главный экран не должен полностью перестраиваться после: изменения корзины; изменения домашних запасов; открытия продавца; открытия товара. Обновляются только изменившиеся области интерфейса.

## 18. Критерии приёмки

Главный экран считается реализованным, если: карта остаётся главным рабочим пространством; Bottom Sheet управляет всеми сценариями; отсутствуют полноэкранные переходы между пользовательскими сценариями; состояние карты сохраняется при любых действиях; Back работает по стеку Bottom Sheet; интерфейс остаётся отзывчивым во время фоновых операций; все пользовательские сценарии из ТЗ-006 выполняются без смены главного экрана.
