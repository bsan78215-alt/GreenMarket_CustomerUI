## Назначение

Карточка продавца предназначена не для просмотра всего ассортимента продавца, а для принятия решения: «Стоит ли включать этого продавца в текущую покупку?»

Поэтому вся информация на экране отображается относительно текущего Purchase Intent.

## Открытие карточки

Карточка открывается: с карты; из варианта покупки; из поиска; из списка любимых продавцов.

## Верхняя часть

Отображается краткая информация. Например: название продавца; фотография; рейтинг; расстояние; время работы; является ли любимым.

## Выполнение текущей покупки

Это главный блок карточки. Например:

```
Текущая покупка
✓ Домашняя колбаса
✓ Сыр
✓ Огурцы
✓ Помидоры
✗ Пиво отсутствует

Выполнение покупки
4 из 5 товаров
```

Именно этот блок должен занимать первое место. Не описание продавца. Не отзывы.

## Предлагаемые замены

Если какого-то товара нет. Например: «Домашняя колбаса отсутствует. Продавец рекомендует ✓ Колбаса сыровяленая», или «Редиски сегодня нет. Есть молодой шпинат.»

Пользователь может: принять замену; отказаться; отложить решение. (В будущем это может быть согласовано с другим участником семьи.)

## Остальной каталог

Только после товаров из текущей покупки. Раздел: «Другие товары продавца».

## Качество товаров

Для каждого товара могут отображаться: свежесть; сезонность; рекомендации продавца; фотографии; сертификаты (если есть).

## Взаимодействие с продавцом

Пользователь может: добавить в любимые; убрать из любимых; открыть маршрут; посмотреть все товары; сообщить о неточности информации.

## Работа с корзиной

Из карточки продавца пользователь может: добавить товар; удалить товар; заменить товар; изменить количество.

После любого изменения: Purchase Intent пересчитывается; вариант покупки обновляется; карта обновляется автоматически.

## Возврат

После закрытия карточки продавца пользователь возвращается: к карте; с тем же масштабом; с тем же выбранным вариантом покупки; с тем же положением Bottom Sheet.

## Достоверность информации

Раздел, добавленный отдельно как важный для GreenMarket.

У каждого товара должна быть отметка: когда информация обновлялась; кем обновлялась (продавцом / системой / покупателем / оператором); уровень доверия к информации.

Например:

```
Домашняя колбаса
Наличие подтверждено
12 минут назад
Источник: ✓ продавец
```

```
Редиска
Последняя проверка
3 часа назад
Источник: ✓ покупатель
```

---

**Комментарий из чата:** это напрямую связано с одной из ключевых идей — покупатель тоже становится источником актуальной информации с рынка. Даже если подтверждение пользователями появится позже, карточку продавца стоит проектировать так, чтобы в ней уже было место для отображения происхождения и свежести данных. Это повысит доверие к информации и позволит постепенно развивать механизм коллективной актуализации ассортимента без изменения структуры интерфейса.
