## Назначение

Customer UI представляет собой конечный автомат (FSM), в котором пользователь всегда находится в одном из состояний.

Переходы между состояниями инициируются действиями пользователя или событиями системы.

## State 1. Exploring Market

Первое состояние после запуска приложения.

**Карта.** Показывает: любимых продавцов; ближайшие точки; точки возле сохранённых мест; точки вдоль регулярных маршрутов.

**Bottom Sheet.** Показывает: текущую корзину; домашние запасы; типовые корзины.

**Возможные действия:** изменить корзину; выбрать типовую корзину; открыть продавца; открыть товар; поиск товара.

## State 2. Editing Purchase

Пользователь изменяет покупку. Например: добавляет товар; удаляет товар; меняет количество; изменяет домашние запасы.

Карта остаётся видимой. Варианты покупки становятся неактуальными.

## State 3. Calculating Purchase

Активируется после команды «Подобрать покупку».

В это время: пользователь продолжает работать с интерфейсом; карта не блокируется; отображается индикатор расчёта.

После завершения происходит переход.

## State 4. Viewing Purchase Options

Bottom Sheet показывает найденные варианты. Например: ⭐ Рекомендуемый; ❤️ Любимые продавцы; 💰 Минимальная цена; 🚶 Минимальный маршрут.

Карта подсвечивает выбранный вариант.

## State 5. Viewing Seller

Bottom Sheet показывает карточку продавца. Карта продолжает отображать: положение продавца; остальные точки выбранного варианта.

## State 6. Viewing Product

Bottom Sheet показывает карточку товара. Карта не изменяется.

## State 7. Shopping Route

Пользователь начал выполнение выбранного варианта.

Карта показывает: маршрут; следующего продавца; уже посещённые точки.

Bottom Sheet показывает: текущего продавца; список оставшихся товаров.

## State 8. Purchase Completed

Покупка завершена. Система предлагает: обновить домашние запасы; сохранить изменения корзины; оценить продавцов; подтвердить изменения ассортимента.

После завершения происходит возврат в Exploring Market.

## Правила переходов

```
Exploring Market
  ↓
Editing Purchase
  ↓
Calculating Purchase
  ↓
Viewing Purchase Options
  ↓
Viewing Seller
  ↓
Viewing Product
  ↓
Shopping Route
  ↓
Purchase Completed
  ↓
Exploring Market
```

## Очень важное правило

Любое состояние может быть прервано.

Например: во время просмотра продавца пользователь может изменить корзину; выполнить новый поиск; открыть другой вариант покупки.

FSM должна корректно обработать такой переход без сброса всего интерфейса.
