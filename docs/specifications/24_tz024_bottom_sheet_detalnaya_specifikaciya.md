## Назначение

Bottom Sheet является основной рабочей областью Customer UI.

Пользователь взаимодействует с GreenMarket преимущественно через Bottom Sheet. Карта используется как пространственный контекст.

## 1. Общие принципы

Bottom Sheet не является экраном. Bottom Sheet является контейнером состояний интерфейса. В любой момент времени отображается только одно активное содержимое.

## 2. Структура

Bottom Sheet состоит из пяти частей:

```
┌───────────────────────────────┐
│ Drag Handle                    │
├───────────────────────────────┤
│ Header                         │
├───────────────────────────────┤
│ Toolbar (если требуется)       │
├───────────────────────────────┤
│ Content                        │
├───────────────────────────────┤
│ Bottom Actions                 │
└───────────────────────────────┘
```

## 3. Drag Handle

Всегда отображается. Назначение: показать возможность изменения высоты; обеспечить жест перетаскивания. Размер должен позволять удобную работу одной рукой.

## 4. Header

Header зависит от текущего контекста. Например: «Главный экран» — «Покупка на сегодня, 12 товаров»; «Поиск» — «Поиск»; «Продавец» — «Фермер Иван»; «Товар» — «Домашняя колбаса».

Header никогда не содержит длинные описания.

## 5. Toolbar

Отображается только при необходимости. Например: поиск; фильтры; сортировка; выбор режима.

## 6. Content

Основная рабочая область. Именно Content изменяется при смене состояния:

```
Главный экран → Корзина → Варианты покупки → Карточка продавца → Карточка товара
```

## 7. Bottom Actions

Нижняя панель действий. Содержит только действия текущего контекста. Например: главный экран — «Подобрать покупку»; продавец — «Начать маршрут», «Избранное»; товар — «Добавить», «Заменить».

## 8. Состояния высоты

Используются четыре уровня:

- **Peek** — отображается только Header. Используется редко.
- **Collapsed** — ~25%. Главный режим просмотра карты.
- **Half** — ~55%. Основной рабочий режим.
- **Expanded** — ~90%. Полная работа с карточками.

## 9. Стек содержимого

Bottom Sheet имеет собственную навигацию, например: `Главный экран → Поиск → Продавец → Товар`. Back работает только со стеком Bottom Sheet.

## 10. Карта не входит в стек

При возврате не происходит: пересоздание карты; изменение масштаба; изменение положения камеры. Изменяется только Content Bottom Sheet.

## 11. Замена Content

Содержимое заменяется без закрытия Bottom Sheet. Например: «Поиск → Продавец» — плавная горизонтальная анимация.

## 12. Жесты

Поддерживаются: поднять; опустить; закрыть клавиатуру свайпом вниз. Не допускаются случайные закрытия.

## 13. Работа клавиатуры

При открытии клавиатуры Bottom Sheet автоматически переходит в Expanded. Карта уменьшается только визуально, её состояние сохраняется.

## 14. Прокрутка

Если список достиг верхней границы, следующий жест начинает перемещать Bottom Sheet. Это соответствует Material Design.

## 15. Анимации

Используются только: изменение высоты; замена Content; плавное изменение Header; плавное изменение Bottom Actions.

Не допускаются: полные исчезновения Bottom Sheet.

## 16. Восстановление состояния

После поворота экрана или возврата из фона восстанавливаются: высота Bottom Sheet; содержимое; положение прокрутки; стек; открытая карточка.

## 17. Производительность

При изменении данных не перестраивается весь Bottom Sheet. Обновляется только изменившийся компонент. Например: изменилась цена товара — перерисовывается только карточка товара.

## 18. Пустые состояния

Каждый тип контента имеет собственное Empty State: нет результатов поиска; нет продавцов; нет вариантов покупки; нет товаров; нет домашних запасов.

## 19. Ошибки

Ошибки отображаются внутри Bottom Sheet. Не допускается открытие отдельных экранов ошибок.

## 20. Правила проектирования

Bottom Sheet всегда должен отвечать на три вопроса пользователя: где я сейчас? что я могу сделать дальше? что изменится после моего действия?

Если хотя бы на один вопрос нет очевидного ответа, содержимое Bottom Sheet считается спроектированным неправильно.
