## Назначение

Документ определяет единое значение терминов, используемых в GreenMarket.

Все документы проекта должны использовать только определения из настоящего глоссария.

## Домен пользователя

**Household** — домохозяйство. Совокупность пользователей, совместно ведущих домашнее хозяйство. В MVP может состоять из одного участника.

**User** — участник домохозяйства. Имеет собственные настройки, но работает в контексте домохозяйства.

**Place** — значимое место пользователя. Например: Дом, Работа, Дача, Любимый рынок. Используется при поиске продавцов и построении рекомендаций.

**Regular Route** — регулярный маршрут пользователя. Например: Дом → Работа. Используется при формировании предложений покупки «по пути».

## Покупка

**Basket** — постоянный шаблон покупки. Корзина не является текущей покупкой. Она служит отправной точкой. Примеры: Недельная закупка, На шашлык, Завтрак.

**Basket Item** — элемент корзины. Описывает потребность в товаре.

**Home Inventory** — домашние запасы. Список продуктов, уже имеющихся дома. Используется при формировании текущей покупки.

**Purchase Intent** — текущая покупка. Вычисляется автоматически на основе: корзины; домашних запасов; изменений пользователя; ограничений пользователя. Не хранится как постоянный объект.

**Purchase Option** — один из способов выполнить Purchase Intent. Может отличаться: стоимостью; количеством продавцов; маршрутом; использованием любимых продавцов.

**Shopping Route** — маршрут выполнения выбранного Purchase Option. Определяет последовательность посещения продавцов.

## Продавцы

**Seller** — продавец GreenMarket. Предлагает товары собственного каталога.

**Favorite Seller** — любимый продавец пользователя. Используется системой при подборе вариантов покупки.

**Preferred Seller** — предпочтительный продавец для конкретной категории товаров. Например: творог только у этого продавца. (Не входит в MVP.)

## Каталог

**Catalog** — полный каталог товаров продавца.

**Product** — конкретный товар продавца.

**Product Availability** — информация о наличии товара.

**Product Quality** — совокупность характеристик товара. Например: свежесть; сезонность; происхождение; фотографии; сертификаты.

## Карта

**Map Context** — текущее состояние карты. Например: обзор; покупка; маршрут.

**Market Point** — точка продажи на карте. Не обязательно продавец. В будущем это может быть: палатка; фермерское хозяйство; пункт выдачи.

## Интерфейс

**Bottom Sheet** — основная рабочая область интерфейса. Содержимое зависит от текущего контекста.

**Main Screen** — главный экран приложения. Состоит из карты и Bottom Sheet.

## Алгоритмы

**Purchase Optimizer** — компонент системы, рассчитывающий варианты выполнения Purchase Intent. Он не отображается пользователю. Его результатом являются Purchase Options.

## Очень важный раздел: что НЕ является корзиной

Во многих маркетплейсах корзина — это список товаров перед оплатой. В GreenMarket это разные понятия.

- Basket — постоянный шаблон потребностей пользователя.
- Purchase Intent — сегодняшняя покупка, сформированная на основе Basket.
- Purchase Option — один из способов выполнить Purchase Intent.

Это три разных объекта, которые нельзя смешивать ни в документации, ни в коде.
