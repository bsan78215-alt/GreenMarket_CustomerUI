# ТЗ-025. Карточка продавца (Seller Card)

**Версия:** 1.0
**Статус:** Draft

## 1. Назначение

Карточка продавца предназначена не для описания продавца.

Она помогает пользователю принять решение: покупать ли у этого продавца товары для текущей покупки.

Главным объектом карточки является не продавец, а его вклад в выполнение текущей Purchase Intent.

## 2. Когда открывается карточка

Карточка открывается:
- нажатием на маркер продавца на карте;
- выбором продавца в Purchase Option;
- выбором продавца из результатов поиска;
- переходом из карточки товара.

При открытии карта сохраняет: положение камеры; масштаб; маршрут; выделенные объекты.

Bottom Sheet переходит в состояние Expanded.

## 3. Общая структура

```
┌───────────────────────────────┐
│ Header                        │
├───────────────────────────────┤
│ Purchase Coverage             │
├───────────────────────────────┤
│ Important Alerts              │
├───────────────────────────────┤
│ Seller Information            │
├───────────────────────────────┤
│ Products for Current Basket   │
├───────────────────────────────┤
│ Other Products                │
├───────────────────────────────┤
│ User Reports                  │
├───────────────────────────────┤
│ Bottom Actions                │
└───────────────────────────────┘
```

## 4. Header

Отображается: название продавца; фотография; рейтинг; расстояние; время работы; статус (открыт/закрыт). Длинные описания не отображаются.

## 5. Purchase Coverage

Самый важный блок карточки. Показывает: «Корзина закрывается — 18 из 22 товаров», сколько товаров есть, сколько отсутствует, сколько потребуется купить у других продавцов. Если продавец полностью закрывает корзину — специальное сообщение.

## 6. Important Alerts

Появляется только при наличии информации. Например: «Нет свежей редиски», «Есть молодой шпинат», «Сегодня скидка на сыр».

## 7. Seller Information

Специализация; часы работы; способы оплаты; фотографии точки; уровень доверия информации.

## 8. Products for Current Basket

Главный список — только товары текущей покупки. Для каждого: название, цена, наличие, актуальность информации, количество, признак отсутствия, возможная замена. Показывается первым.

## 9. Other Products

Сворачиваемый раздел, остальные товары продавца. По умолчанию скрыт.

## 10. User Reports

Сообщения покупателей (не было товара, изменилась цена, качество отличное, новые фото), с датой каждого сообщения.

## 11. Bottom Actions

Только действия текущего контекста: Начать маршрут; Добавить в избранное; Сообщить об отсутствии товара; Сообщить об изменении цены; Поделиться продавцом. Набор кнопок приходит через availableActions — UI сам ничего не определяет.

## 12. ViewModel

Карточка работает только с `SellerCardViewModel`: seller, coverage, importantAlerts, basketProducts, otherProducts, trustInfo, availableActions, photos, reports. Вычислений внутри React не допускается.

## 13. Взаимодействие с картой

Карточка не управляет картой напрямую. При выборе продавца отправляется Action. Изменение карты — через Business Events и MapProjection.

## 14. Навигация

Из карточки продавца: к карточке товара; к маршруту; к поиску. Back возвращает к предыдущему содержимому Bottom Sheet. Карта остаётся без изменений.

## 15. Loading

Skeleton-компоненты, пустой экран не допускается. Карта остаётся полностью интерактивной.

## 16. Empty State

«У этого продавца нет товаров из вашей текущей покупки.» При этом доступен просмотр полного каталога.

## 17. Offline

Показываются последние сохранённые данные + предупреждение «Данные могут быть неактуальны».

## 18. Ошибки

Bottom Sheet остаётся открытым, отображается сообщение об ошибке и кнопка повторной загрузки. Отдельный экран ошибки не допускается.

## 19. Производительность

При изменении одного товара не перестраивается вся карточка — обновляется только элемент списка.

## 20. Критерии приёмки

- пользователь за несколько секунд понимает, насколько продавец подходит для текущей покупки;
- список товаров для текущей корзины отображается раньше полного каталога;
- карта не теряет состояния;
- Bottom Sheet не закрывается при переходах;
- React не содержит бизнес-логики;
- все действия — через dispatch(Action);
- UI использует только SellerCardViewModel и availableActions.

## Архитектурное замечание

Ключевое отличие от классических маркетплейсов: карточка отвечает не на вопрос «Кто этот продавец?», а на вопрос «Насколько этот продавец помогает выполнить мою текущую покупку?». Поэтому Purchase Coverage расположен выше описания продавца и каталога — в центре внимания покупка пользователя, а не продавец сам по себе. Отличие стоит сохранить во всех последующих экранах.
