## 1. Назначение

Карточка продавца помогает пользователю оценить, насколько данный продавец подходит для выполнения текущей Purchase Intent.

Карточка не является справочником о продавце.

Главный вопрос, на который она отвечает:

> Стоит ли включить этого продавца в текущий вариант покупки?

## 2. Общая структура

Порядок блоков является обязательным.

```
Header
↓
Purchase Coverage
↓
Important Alerts
↓
Basket Products
↓
Seller Information
↓
Other Products
↓
User Reports
↓
Bottom Actions
```

## 3. Header

Header отображается всегда.

Содержит:

- фотографию;
- название;
- рейтинг;
- расстояние;
- статус работы.

### Sticky Header

При прокрутке Header остаётся закреплённым.

Допускается переход в компактный режим:

- уменьшается высота;
- фотография уменьшается;
- основные действия остаются доступными.

## 4. Purchase Coverage

Главный блок карточки.

Отображает:

- количество найденных товаров;
- процент покрытия корзины;
- количество отсутствующих товаров;
- предполагаемую стоимость покупки у данного продавца.

Если продавец полностью покрывает корзину, отображается соответствующий индикатор.

## 5. Important Alerts

Показываются только значимые изменения.

Например:

- отсутствует сезонный товар;
- появилась рекомендуемая замена;
- изменились цены;
- появились новые пользовательские сообщения.

При отсутствии предупреждений раздел не отображается.

## 6. Basket Products

Основной список карточки.

Отображаются только товары из текущей Purchase Intent.

Порядок элементов фиксирован:

1. Доступные товары.
2. Предлагаемые замены.
3. Отсутствующие товары.

Каждый товар отображает:

- название;
- цену;
- наличие;
- количество;
- актуальность информации;
- возможную замену.

### Производительность

Список должен поддерживать виртуализацию.

При изменении одного товара обновляется только соответствующий элемент списка.

## 7. Seller Information

Отображаются:

- специализация;
- режим работы;
- способы оплаты;
- уровень доверия;
- дата последнего подтверждения информации.

Если данные устарели, отображается индикатор:

> Информация может быть неактуальной.

## 8. Other Products

Раздел содержит остальные товары продавца.

По умолчанию свернут.

Подгружается по требованию пользователя.

## 9. User Reports

Показываются последние сообщения покупателей.

Каждое сообщение содержит:

- текст;
- автора (если доступно);
- дату;
- уровень доверия.

## 10. Фотографии

Фотографии загружаются лениво (lazy loading).

При отсутствии изображения отображается стандартная заглушка.

Загрузка фотографий не должна блокировать отображение карточки.

## 11. Bottom Actions

Карточка отображает только действия, полученные в составе `availableActions`.

React не вычисляет доступность действий.

Все действия отправляются через:

```
dispatch(Action)
```

## 12. ViewModel

Карточка использует исключительно `SellerCardViewModel`.

Вычисления внутри React запрещены.

Все данные должны приходить подготовленными.

## 13. Работа с картой

Карточка не взаимодействует с картой напрямую.

Она не вызывает:

- изменение камеры;
- построение маршрута;
- подсветку продавцов.

Все изменения карты происходят через Platform Core и `MapProjection`.

## 14. Навигация

Поддерживаются переходы:

- к карточке товара;
- к маршруту;
- к поиску.

Кнопка Back работает только со стеком Bottom Sheet.

Карта сохраняет своё состояние.

## 15. Loading

Во время загрузки отображаются skeleton-компоненты.

Пользователь может продолжать взаимодействовать с картой.

## 16. Empty State

Если товары для текущей Purchase Intent отсутствуют, отображается сообщение:

> У данного продавца нет товаров для вашей текущей покупки.

При этом остаётся доступен просмотр полного каталога.

## 17. Offline

Используются последние сохранённые данные.

Отображается предупреждение о возможной неактуальности информации.

## 18. Ошибки

Ошибки отображаются внутри Bottom Sheet.

Отдельный экран ошибок не открывается.

## 19. Критерии приёмки

Карточка считается реализованной, если:

- главным блоком является Purchase Coverage;
- список товаров текущей покупки отображается раньше полного каталога;
- Header остаётся закреплённым при прокрутке;
- большие списки работают с виртуализацией;
- фотографии загружаются независимо от остального содержимого;
- React не содержит бизнес-логики;
- все действия выполняются через `dispatch(Action)`;
- карта изменяется только через `MapProjection`;
- карточка использует только `SellerCardViewModel` и `availableActions`.
